# M1 Spec: 工程与连接闭环

## 目标

实现 Mac App 到 `cc-agent-daemon` 的可靠连接闭环：输入 WS 配置和 token、认证、进入业务 shell，并在非主动断线后自动重连。

## 用户故事

- 作为用户，我可以输入 daemon 地址和 token 连接本机或局域网 daemon。
- 作为用户，我能看到连接中、连接成功、连接失败的明确状态。
- 作为用户，daemon 短暂断开后 App 能自动重连，不清空我的地址和 token。
- 作为开发者，我能通过单测验证 WS URL、JSON-RPC 和重连退避。

## 范围

- App Shell 和连接状态门禁。
- 登录页。
- Keychain token 存储。
- UserDefaults WS 配置存储。
- `URLSessionWebSocketTask` JSON-RPC client。
- daemon notification 接收入口。
- 非主动断开指数退避重连。

## 非范围

- 不要求会话列表完整展示。
- 不要求聊天功能。
- 不要求权限交互。
- 不启动或管理 daemon 进程。

## 功能需求

### 登录页

- 提供 host、port、TLS、token 输入。
- 提供 token 显示/隐藏切换。
- 提供连接按钮。
- 连接中禁用重复提交并展示 loading。
- 连接失败展示可读错误。
- 连接成功进入业务 shell。

### 本地存储

- token 必须写入 Keychain。
- host、port、TLS 写入 UserDefaults。
- 切换连接或主动断开不清空 token 和 WS 配置。
- 错误提示和日志不得展示完整带 token URL。

### WS URL

- 支持 `ws://host:port/ws?token=...`。
- TLS 开启时使用 `wss`。
- token 必须 URL encode。
- token 为空时不追加 token query，但局域网无认证模式不作为默认推荐路径。

### JSON-RPC

- request 使用递增 id。
- response 通过 id 匹配 pending continuation。
- `error` response 转成可读错误。
- 带 `method` 的消息进入 notification handler。
- pending RPC 在连接关闭时全部失败并清理。

### 认证

- WebSocket open 后执行 `auth`。
- `auth` 成功才进入 connected。
- 可选执行 `ping` 作为健康检查。

### 重连

- 主动断开不重连。
- 非主动断开进入 connecting 状态。
- 重连延迟为 `min(1000 * 2^attempt, 30000)` ms。
- 重连成功后触发 `reconnectNonce` 或等价事件，通知业务页重新拉取状态。
- 重连期间保留当前 route 和本地偏好。

## 状态定义

- `disconnected`：未连接或主动断开。
- `connecting`：首次连接或重连中。
- `connected`：认证成功，可执行 RPC。

## RPC 依赖

- `auth`
- `ping` 可选

## UI 要求

- 登录页符合 macOS 原生表单体验。
- 长 host/token 不撑破布局。
- 错误文案应指向可操作原因，例如认证失败、无法连接、连接关闭。

## 验收标准

- 输入 `127.0.0.1:4733` 和正确 token 后可连接 daemon 并进入业务 shell。
- token 错误时停留登录页并显示认证失败。
- daemon 不可达时显示连接失败，不崩溃。
- 主动断开后状态为 disconnected，不重连。
- 非主动断开后状态为 connecting，并按指数退避重连。
- daemon 恢复后自动 connected，并触发业务刷新信号。
- token 不出现在普通日志、错误提示或 UI URL 中。

## 测试要求

- WS URL 构造和 token encoding。
- JSON-RPC success/error/notification decode。
- pending RPC 在 close 时失败。
- 重连退避最大 30 秒。
- 主动断开不重连，非主动断开重连。

## 风险

| 风险 | 影响 | 处理 |
|---|---|---|
| `URLSessionWebSocketTask` 没有显式 open 回调 | auth 时机不稳定 | resume 后启动 receive loop，并用首个 RPC 失败处理兜底 |
| 重连后业务页状态过期 | 当前会话收不到事件 | 通过 reconnect nonce 驱动 M2/M3 重新拉取 |
| token 泄漏 | 安全风险 | 统一 URL 脱敏和错误脱敏 |
