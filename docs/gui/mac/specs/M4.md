# M4 Spec: Web 等价能力

## 目标

补齐 Web 端已验证的关键能力，使 Mac 聊天体验达到 PRD 主体验收：历史与实时消息样式一致、权限交互完整、AskUserQuestion 可用、session alias 不丢事件、聊天侧栏和偏好可用。

## 用户故事

- 作为用户，我重新打开历史会话时，assistant 回复、thinking、工具调用和实时输出样式一致。
- 作为用户，我可以编辑工具权限的 `updatedInput`，也可以填写拒绝原因。
- 作为用户，遇到 AskUserQuestion 时可以通过选项完成回答。
- 作为用户，我可以在聊天页侧栏切换会话，并保留自动跟随等偏好。
- 作为用户，resume 或重连后不会丢失当前会话事件。

## 范围

- 完整历史消息转换。
- Tool result 合并。
- Session alias registry。
- 完整权限响应。
- AskUserQuestion 解析和 UI。
- 聊天侧栏和偏好持久化。
- 自定义模型和设置对齐。

## 非范围

- 不做长会话窗口化性能优化；M5 实现。
- 不做签名和发布。
- 不做 App 内 daemon 管理。

## 功能需求

### 历史消息转换

必须把历史 JSONL 转换为统一 `ChatMessage` / `MessageBlock`：

- user 文本消息渲染为用户气泡。
- assistant content block 渲染为结构化 blocks。
- text block 使用 Markdown 渲染。
- thinking block 可折叠。
- tool_use block 渲染为工具卡。
- metrics 包含 model、token usage、elapsed time。

必须过滤：

- compact summary。
- transcript-only。
- `compact_boundary`。
- 纯 tool_result user message。

### Tool result 合并

- 从历史 user tool_result 中提取 `tool_use_id`。
- 更新对应 tool_use 卡片状态为 completed 或 error。
- 工具结果内容可展开查看。
- 历史 tool_result 和实时 tool_result 使用同一 `ToolResultState`。

### 实时消息一致性

- `stream_event` 和完整 assistant snapshot 都进入同一 message pipeline。
- live tool_use 初始 pending。
- result 或 completed status 结束 streaming。
- metrics 需要从 message_start、message_delta、result 中合并。

### Session alias

- 维护 disk sessionId、runtimeId、sdkSessionId 的 alias set。
- `system init` 返回新 session id 时注册 alias。
- `session.resume` 返回新 live session id 时注册 alias。
- notification 任一 id 命中 alias 即归属当前会话。
- 运行状态更新应同步到 alias 集合内所有 id。

### 权限响应

普通权限弹窗必须支持：

- 展示工具名。
- 展示并允许编辑 JSON object 格式的 `updatedInput`。
- allow 时提交编辑后的 `updatedInput`。
- deny 时提交拒绝原因 message。
- JSON 非 object 时阻止提交并展示错误。

### AskUserQuestion

- 当 `permission/request.toolName == "AskUserQuestion"` 时使用专用选择器。
- 从 `input.questions` 解析问题。
- 支持单选和多选。
- 每个问题至少选择一个选项后才允许提交。
- 提交时构造 `{ ...rawInput, answers }`，answers 以 question 文本为 key。
- 用户取消时 deny，并提交明确 message。

### 聊天侧栏

- 聊天页左侧展示全部工作区和会话。
- 支持收起/展开。
- 当前会话有选中态。
- 会话运行态 badge 与首页一致。
- 支持从侧栏进入新对话和历史会话。
- 侧栏展开状态持久化。

### 自动跟随

- 支持自动跟随回复开关。
- 开启时 streaming 输出自动滚动到底部。
- 用户手动向上滚动时可暂停跟随，具体行为可参考 Web。
- 偏好持久化。

### 模型设置

- `settings.get` 返回的默认模型、effort、permission mode 应作为初始值。
- 支持 Sonnet、Opus、Haiku、自定义模型。
- 历史会话切换模型或 effort 时，按 Web 行为 resume 当前历史 session。

## RPC 依赖

- M3 全部 RPC。
- `settings.get`
- `session.listActive`
- `permission.respond`

## UI 要求

- 历史和实时消息不能出现两套明显不同样式。
- 权限弹窗不裸露为纯 JSON，需要有工具名、行为按钮和错误提示。
- AskUserQuestion 是选项式 UI，不要求用户手写 JSON。
- 聊天侧栏长路径和 sessionId 必须截断。

## 验收标准

- 同一会话实时输出后重开，text、thinking、tool_use、tool_result 样式一致。
- 历史 tool_result 能正确回填到对应 tool_use 卡片。
- compact summary、transcript-only、纯 tool_result user message 不展示为普通对话。
- resume、attachIfLive、system init 后事件不丢失。
- 普通权限 allow 可提交编辑后的 `updatedInput`。
- 普通权限 deny 可提交拒绝原因。
- AskUserQuestion 支持单选/多选，并写回 answers。
- 聊天侧栏可收起，选中态和运行态正确。
- 自动跟随偏好可切换并持久化。

## 测试要求

- `HistoryMessageConverter` 覆盖 text、thinking、tool_use、tool_result。
- compact/transcript-only/filter 规则。
- metrics merge。
- tool result merger。
- alias registry matching。
- permission response builder。
- AskUserQuestion parser、toggle selection、answers builder。
- chat sidebar preference store。
- model selection/custom model 状态逻辑。

## 风险

| 风险 | 影响 | 处理 |
|---|---|---|
| Swift 和 Web 历史转换逻辑漂移 | 样式和行为不一致 | 以 Web 测试用例为蓝本补 Swift 单测 |
| 权限 JSON 编辑体验差 | 用户无法安全修改 input | M4 先保证 object 校验和错误提示，M5 再优化编辑器 |
| alias 过宽误收其他会话事件 | 消息串会话 | alias 只由 create/resume/attach/init 明确关系注册 |
