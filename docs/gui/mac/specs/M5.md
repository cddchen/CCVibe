# M5 Spec: 性能、设计和发布准备

## 目标

完成 Mac App 发布前的体验和工程门禁：长会话滚动性能、Markdown QA、视觉降级、主题、universal build、Hardened Runtime 和签名预留。

## 用户故事

- 作为用户，我打开长历史会话时滚动仍然流畅。
- 作为用户，我能正常阅读 Markdown 表格、代码块、图片和链接。
- 作为用户，App 能跟随系统 light/dark 主题，也能手动切换。
- 作为发布负责人，我能构建 Intel + Apple Silicon 可发布产物。

## 范围

- 长会话窗口化或分页。
- Markdown 表格、代码块、图片、链接 QA。
- 工具结果大文本展示优化。
- Material / `NSVisualEffectView` 统一样式。
- light/dark/system 主题。
- universal build 验证。
- Hardened Runtime、entitlements、签名、notarization 预留。

## 非范围

- 不要求真正接入未来 Liquid Glass 新 API。
- 不要求完成 notarization 自动化流水线。
- 不要求 Mac App 内启动/管理 daemon，除非产品另行决策。

## 功能需求

### 长会话性能

- 消息列表初始可使用 `ScrollView + LazyVStack + ScrollViewReader`。
- 超过阈值的长会话必须采用窗口化或分页策略。
- 初始渲染可只展示最近 N 条，顶部提供加载更早消息。
- 流式更新只 patch 当前 assistant turn，不重建历史消息数组。
- Markdown 解析结果应可缓存，避免历史消息在每次 delta 时重渲染。

### Markdown QA

- text block 支持 Markdown 基础语法。
- 代码块可选中文本，长代码横向滚动或不撑破容器。
- 表格可读，宽表格横向滚动或自适应。
- 图片可以加载并限制最大宽高。
- 链接可点击，外部打开策略符合 macOS 习惯。
- 如果 MarkdownUI 表格能力不足，必须通过 `MarkdownBlock` 替换点局部替换 renderer。

### 工具结果展示

- 大型工具输入和结果限制最大高度。
- 内容区域内部滚动。
- error 状态有明确视觉区分。
- JSON 输入格式化展示。

### 视觉和主题

- 提供 light、dark、system 三种主题偏好。
- 主题偏好持久化。
- 统一 `glassCard` 或等价 modifier。
- macOS 14/15 使用 Material / `NSVisualEffectView`。
- 不直接引用当前 SDK 不存在的 Liquid Glass API。

### 发布工程

- App target 开启 Hardened Runtime。
- entitlements 最小化。
- 明确 bundle id、version、build number 来源。
- 验证 arm64 和 x86_64 构建路径。
- README 记录 build/test/package 基本命令。

## UI 要求

- 缩放窗口时列表、composer、侧栏不重叠。
- 代码块、表格、工具输入、sessionId、路径不撑破布局。
- 主题切换即时生效。
- Material 样式不能影响文本可读性。

## 验收标准

- 真实长历史会话滚动无明显卡顿。
- 流式输出期间历史消息不反复闪烁或重排。
- Markdown 代码块、表格、图片、链接均可读可操作。
- 工具结果大文本不会撑破消息区域。
- light/dark/system 主题可切换并持久化。
- macOS 14/15 构建不依赖 Liquid Glass 新 API。
- `xcodebuild -scheme CCAgent -destination 'platform=macOS' test` 通过，或明确记录环境阻塞。
- Intel + Apple Silicon universal build 路径明确。

## 测试要求

- 长消息列表渲染基准或手动 QA 记录。
- Markdown fixture QA：代码块、表格、图片、链接。
- 主题 preference store 单测。
- 工具结果大文本布局检查。
- build/test 命令记录。

## 风险

| 风险 | 影响 | 处理 |
|---|---|---|
| SwiftUI 长列表仍卡顿 | 长会话不可用 | 启用窗口化，必要时桥接 AppKit 列表 |
| MarkdownUI 表格不足 | PRD Markdown 验收失败 | 保持 `MarkdownBlock` 替换点，局部使用自定义 renderer |
| 视觉材质降低可读性 | 深浅色体验差 | 主题 QA 必须覆盖高对比文本 |
| universal build 环境缺失 | 发布阻塞 | M5 记录命令和阻塞项，提前暴露 |
