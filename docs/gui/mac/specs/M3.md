# M3 Spec: 聊天 MVP

## 目标

实现聊天页最小可用闭环：进入工作区、信任校验、加载历史、新建或续聊会话、发送消息、接收实时输出、停止运行。

## 用户故事

- 作为用户，我可以从首页选择目录开始新对话。
- 作为用户，我可以打开历史会话并继续追问。
- 作为用户，如果会话仍在运行，我能挂接并继续看到输出。
- 作为用户，我可以在运行中停止当前会话。
- 作为用户，我可以选择模型、思考强度和权限模式。

## 范围

- Chat route 和 ChatViewModel。
- 工作区信任弹窗。
- 基础历史加载。
- 会话 create/attach/resume/send/interrupt。
- 实时 text、thinking、tool_use 渲染。
- 基础模型、effort、permission mode 控件。
- 普通权限 allow/deny 的最低能力。

## 非范围

- 不要求历史 assistant/tool 完整转换；M4 补齐。
- 不要求 AskUserQuestion；M4 实现。
- 不要求聊天侧栏；M4 实现。
- 不要求长会话窗口化；M5 实现。

## 功能需求

### 路由

- 支持新对话 route：`chat(workspacePath, nil)`。
- 支持历史会话 route：`chat(workspacePath, sessionId)`。
- 新建会话成功后 native route 应更新为带 sessionId 的状态，保证窗口标题、侧栏选中态和刷新恢复一致。

### 工作区信任

- 进入聊天后先调用 `workspace.checkTrust`。
- 未信任时展示 modal。
- modal 提供信任当前目录和信任父目录操作。
- 信任前不得调用 `history.loadSession`、`session.create`、`session.resume`。

### 历史加载

- 历史会话进入时调用 `history.loadSession`。
- M3 可先渲染 user 文本和基础 assistant 文本，但不得展示 compact summary、transcript-only 等明显非对话内容。
- 历史加载失败展示错误。

### 实时挂接

- 历史会话加载后调用 `session.attachIfLive`。
- 如果 attached，更新 live session id 和运行状态。
- 如果未 attached，保留历史 session id，后续发送时走 attach 或 resume。

### 发送消息

- Enter 发送，Shift+Enter 换行。
- 空白输入不可发送。
- 运行中不可重复发送，除非后续明确支持队列。
- 无 live session 时调用 `session.create`。
- 有 live session 时优先 `session.attach`。
- `attach` 失败且存在历史 session id 时调用 `session.resume`，再 `session.sendMessage`。
- 发送后追加用户消息，并创建 streaming assistant turn。

### 停止运行

- 运行中显示停止按钮。
- 点击后调用 `session.interrupt`。
- 成功后结束 streaming turn，状态显示已停止或 interrupted。

### 实时渲染

- text delta 追加到 assistant text block。
- thinking delta 追加到 thinking block，流式阶段默认展开。
- tool_use start 创建工具卡片，状态为 pending。
- result 或 completed status 结束当前 turn。
- 不直接把 SDK 原始 JSON 裸露给用户。

### 控件

- 模型：Sonnet、Opus、Haiku、自定义模型入口可在 M4 完善。
- effort：low、medium、high、xhigh、max。
- permission mode：default、acceptEdits、plan、auto、bypassPermissions、dontAsk。
- 切换 permission mode 时，如已有 live session，调用 `session.setPermissionMode`。

## RPC 依赖

- `workspace.checkTrust`
- `workspace.add`
- `history.loadSession`
- `session.create`
- `session.attachIfLive`
- `session.attach`
- `session.resume`
- `session.sendMessage`
- `session.interrupt`
- `session.setPermissionMode`
- `settings.get`
- `permission.respond` 最低 allow/deny

## Notification 依赖

- `session/event`
- `session/status`
- `permission/request`

## UI 要求

- 聊天页由消息区和底部 composer 构成。
- composer 支持多行输入。
- 运行中有明确 busy 状态。
- 错误状态不覆盖用户已输入历史。
- thinking、tool_use、权限弹窗具备可理解标签和状态。

## 验收标准

- 能从目录中新建对话并发送首条消息。
- 新建成功后当前 route/state 绑定 sessionId。
- 能打开历史会话并继续发送消息。
- 如果会话仍在运行，能挂接并继续接收实时输出。
- 运行中停止按钮生效。
- 模型、effort、permission mode 控件可见并能影响 create/resume 参数。
- thinking 和 tool_use 不以原始 JSON 直接裸露。
- 工作区未信任时不能读取历史或创建会话。

## 测试要求

- 工作区信任状态机。
- `ensureSession` create/attach/resume 分支。
- sendMessage attach 失败后的 resume fallback。
- stream event 到 message block。
- status 到 busy/interrupted/completed 状态映射。
- permission allow/deny 最低参数构造。

## 风险

| 风险 | 影响 | 处理 |
|---|---|---|
| history sessionId、runtimeId、sdkSessionId 不一致 | 实时事件丢失 | M3 做基础匹配，M4 实现完整 alias |
| 历史渲染不完整 | 与实时样式不一致 | M3 明确为 MVP，M4 作为退出门槛补齐 |
| 发送失败后 UI 已追加用户消息 | 体验不一致 | 保留消息并展示失败状态，后续可支持重试 |
