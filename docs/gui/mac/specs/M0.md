# M0 Spec: 工程恢复与基线确认

## 目标

恢复或重建 `repos/cc-agent-mac` 原生 SwiftUI 工程，使后续里程碑可以在稳定、可构建、可测试的工程基线上推进。

## 背景

Git 历史中曾存在 `repos/cc-agent-mac` SwiftUI 骨架，但当前工作树里该目录处于删除状态。M0 不追求产品功能完整，而是恢复工程结构、依赖、构建入口和差异清单，避免后续开发在不可构建状态上叠加。

## 用户故事

- 作为开发者，我可以在本地生成 Xcode 工程并打开 Mac App target。
- 作为开发者，我可以运行基础 build/test 命令，快速判断工程是否健康。
- 作为产品和研发协作者，我可以看到当前 Mac 骨架相对 Web 基准还缺哪些能力。

## 范围

- 恢复或重建 `repos/cc-agent-mac` 目录。
- 保留 XcodeGen 作为工程生成方式。
- 确认 app target、unit test target、entitlements、Info.plist、MarkdownUI 依赖。
- 移除不应作为源码基线维护的 Xcode 用户态文件。
- 建立 Mac 端待移植模块清单。

## 非范围

- 不要求完成 daemon 连接。
- 不要求实现可用 UI 流程。
- 不处理签名、notarization、发布包。
- 不接入 Liquid Glass 新 API。

## 功能需求

### 工程结构

- `repos/cc-agent-mac/project.yml` 必须存在。
- `Sources/` 至少包含 App、Auth、Networking、Design、Features、Domain 或等价分层目录。
- `Tests/CCAgentTests/` 必须存在。
- 生成的 `.xcodeproj` 不作为主要维护对象；源头以 `project.yml` 为准。

### 平台与依赖

- 最低支持 macOS 14。
- 支持 Intel 和 Apple Silicon 构建路径。
- Swift 版本采用历史骨架或当前 Xcode 稳定支持版本。
- Markdown MVP 依赖 MarkdownUI，但必须封装在消息组件之后，避免上层强耦合。

### 源码卫生

- 不提交 `xcuserdata`、`UserInterfaceState.xcuserstate` 等用户态文件。
- `Package.resolved` 是否提交由工程约定决定；若保留，需要说明原因。
- README 或 spec 里记录标准 build/test 命令。

### 差异清单

必须记录 Web 端需要移植或对齐的核心模块：

- `daemonClient`
- `messageBlocks`
- `permissionResponses`
- `askUserQuestion`
- `chatSessionRouting`
- `sessionListCache`
- `useActiveSessions`
- `uiPreferences`

## 交付物

- `repos/cc-agent-mac/project.yml`
- `repos/cc-agent-mac/Sources/...`
- `repos/cc-agent-mac/Tests/...`
- `repos/cc-agent-mac/README.md`
- Mac 端差异/待办清单，可放在 README、实现计划或 issue 列表中。

## 验收标准

- `repos/cc-agent-mac/project.yml` 存在。
- `xcodegen generate` 可以成功生成工程，或失败原因是明确的本机环境缺失。
- `xcodebuild -scheme CCAgent -destination 'platform=macOS' build` 可以成功，或只剩明确外部环境阻塞。
- `git status` 不再显示 `repos/cc-agent-mac` 整目录删除。
- 工程中不存在明显不应提交的 Xcode 用户态状态文件。
- 已形成 Web 等价能力待移植清单。

## 测试要求

- 至少保留或新增一个能被 test target 执行的单测。
- 若无法运行 Xcode 命令，必须记录命令、失败原因和是否为环境阻塞。

## 风险

| 风险 | 影响 | 处理 |
|---|---|---|
| 历史骨架恢复后与当前 daemon/Web 不一致 | 后续功能返工 | M0 只恢复基线，M1-M4 再逐项对齐 |
| XcodeGen 未安装 | 无法生成工程 | 记录环境阻塞，不改用手工工程作为长期源头 |
| 误提交 Xcode 用户态文件 | 工程噪声和冲突 | M0 清理并加入 ignore 规则 |
