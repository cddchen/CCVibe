# M2 Spec: 会话首页

## 目标

实现 Mac App 会话首页，按工作目录组织历史和手动添加的工作区，为新建对话、打开历史会话和查看运行状态提供稳定入口。

## 用户故事

- 作为用户，我能看到本机 Claude 历史会话按工作目录分组展示。
- 作为用户，我能手动添加工作目录并从该目录开始新对话。
- 作为用户，我能看到哪些会话仍在运行或可挂接。
- 作为用户，我可以刷新列表、折叠目录、切换连接。

## 范围

- 首页主视图。
- 会话列表聚合服务。
- 工作区添加。
- 目录展开状态持久化。
- active session 轮询和状态标识。

## 非范围

- 不实现聊天消息发送。
- 不实现完整权限交互。
- 不实现 Finder 拖拽添加目录。
- 不要求虚拟化会话列表，除非数据量导致明显卡顿。

## 功能需求

### 数据聚合

首页必须合并以下数据源：

- `history.listAllLocal`：读取本机 Claude 历史项目和会话。
- `workspace.list`：读取手动添加或信任的工作区。
- `history.listSessions`：对需要补齐历史的工作区读取会话。
- `session.listActive`：读取 daemon 当前活跃会话。

### 分组与排序

- 按 `workspacePath` 分组。
- 目录按该目录内最近会话时间倒序。
- 目录内 session 按 `lastTimestamp` 倒序。
- 手动添加但没有历史会话的目录仍可显示，并提供新对话入口。
- sessionId 展示摘要，不直接用超长文本撑开布局。

### 工作区添加

- 支持手动输入目录路径。
- 调用 `workspace.add`。
- 添加成功后清理列表缓存并刷新。
- 添加失败展示错误。

### 展开折叠

- 每个目录可以展开/折叠。
- 展开状态按 workspace path 持久化到 UserDefaults。
- 默认策略：有活跃会话或最近会话的目录优先展开；具体实现可保持简单，但必须稳定。

### Active 状态

- 轮询 `session.listActive`。
- 显示 running、starting、attachable 或等价状态标识。
- 重连成功后立即刷新 active 状态。

### 顶部操作

- 显示连接状态。
- 提供刷新按钮。
- 提供切换连接或断开按钮。
- 主题切换可占位到 M5，但入口不应阻塞 M2。

## RPC 依赖

- `history.listAllLocal`
- `workspace.list`
- `workspace.add`
- `history.listSessions`
- `session.listActive`

## UI 要求

- 使用 macOS 原生列表/侧栏风格。
- 目录路径、sessionId、时间、消息数信息清晰可扫读。
- 长路径必须截断或可横向查看，不撑破窗口。
- 空状态需要说明“暂无历史会话”并提供添加工作区入口。
- loading、error、empty、loaded 状态必须区分。

## 数据模型建议

- `SessionListData`
- `SessionGroup`
- `SessionRow`
- `WorkspaceRow`
- `ActiveSessionState`
- `DirectoryExpansionStore`

## 验收标准

- 首页能展示 `~/.claude/projects` 中已有历史会话。
- 会话按工作目录分组。
- 工作目录按最近会话时间倒序。
- 目录内会话按最近更新时间倒序。
- 手动添加工作区后可刷新并展示。
- 每个目录下有“新对话”入口。
- running/attachable 状态能显示并随轮询更新。
- 长路径和 sessionId 不撑破 UI。
- 重连成功后列表和 active 状态可恢复。

## 测试要求

- `SessionListService` 合并历史和工作区。
- 目录排序。
- session 排序。
- 空工作区保留。
- active 状态映射。
- 展开状态读写。

## 风险

| 风险 | 影响 | 处理 |
|---|---|---|
| 历史目录和手动工作区路径格式不一致 | 分组重复 | 使用 canonical path 或 daemon 返回 path 为准 |
| 会话数量过多 | 首页卡顿 | M2 保持轻量渲染，必要时后续分页 |
| active session id 与 disk id 不一致 | badge 不准 | M4 alias 完整处理，M2 先按 daemon 返回字段映射 |
